# ae Optics CRM — ملخص التعديلات v4.1

## الملفات المعدَّلة

### `src/components/OrderForm.jsx` ← أهم ملف في هذا الإصدار
**التعديلات:**
1. **إصلاح ظهور العملاء** — جلب العملاء مباشرة من Supabase عند فتح الفورم مع Autocomplete بالاسم أو الهاتف
2. **دعم رقمين هاتف** — إضافة حقل `customer_phone2` (الرقم الاحتياطي) في نموذج الطلب
3. **ربط المخزون** — اختيار الفريم من `inventory` + اختيار variant (لون/مقاس) من `inventory_variants` مع عرض الكمية المتاحة
4. **نظام العدسات الاحترافي** — اختيار شركة العدسة ثم variant، جلب السعر تلقائياً
5. **إصلاح الدفع الجزئي** — عند اختيار "جزئي" يظهر حقل "المبلغ المدفوع" ويُحسب المتبقي تلقائياً
6. **دمج الوصفة مع بيانات العميل** — الوصفة الطبية تظهر في نفس قسم العميل في تبويب "الطلب" مباشرة
7. **إخفاء التكاليف الداخلية** — نُقلت للتبويب المخصص "التكاليف"

### `src/pages/DeliveryTracker.jsx`

## ملف قاعدة البيانات الجديد

### `supabase_patch_v4_1.sql`
**يحتوي على:**
- `ALTER TABLE customers ADD COLUMN phone2` — رقم هاتف احتياطي
- `ALTER TABLE orders ADD COLUMN amount_paid` — الدفع الجزئي
- `ALTER TABLE orders ADD COLUMN lens_brand_id, lens_variant_id` — ربط العدسات
- `ALTER TABLE orders ADD COLUMN frame_id, frame_variant_id` — ربط المخزون
- `CREATE FUNCTION get_or_create_customer_v2` — دالة محسّنة مع phone2
- `CREATE FUNCTION deduct_inventory_variant` — خصم المخزون
- `CREATE FUNCTION restore_inventory_variant` — استرداد المخزون
- `CREATE VIEW inventory_with_qty` — عرض المخزون مع الكمية
