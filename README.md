# ae Optics CRM — v1.0.0

نظام إدارة متجر النظارات الأونلاين | مبني بـ React + Supabase

---

## التعديلات المُطبَّقة في هذا الإصدار

### 1 — إصلاح مشكلة Focus في نموذج الطلبات ✅
- نُقلت مكوّنات `Field` و`Inp` و`Sel` خارج الـ component
- لا إعادة render أثناء الكتابة — الكتابة المستمرة تعمل بشكل طبيعي

### 2 — تسجيل تاريخ الورشة تلقائياً ✅
- عند الانتقال إلى **Sent to Workshop**: يُسجَّل التاريخ والوقت تلقائياً
- عند الانتقال إلى **Returned from Workshop**: يُسجَّل التاريخ والوقت تلقائياً
- يعمل من `OrderForm` ومن `QuickUpdate`

### 3 — تبسيط مصادر الطلبات ✅
- المصادر المعتمدة: Website / WhatsApp / Messenger / TikTok / Referral / Repeat Customer
- عند اكتشاف عميل موجود برقم الهاتف → يُعيَّن المصدر تلقائياً `Repeat Customer`

### 4 — إزالة نوع الإطار من شاشة إنشاء الطلب ✅
- لا يظهر حقل frame_type كمحدِّد يدوي
- يُجلَب تلقائياً من بيانات المنتج المختار من المخزون
- يظهر read-only للمراجعة فقط

### 5 — نظام شركات العدسات (Lens Brands) ✅
- صفحة مستقلة `/lens-brands`
- كل شركة تحتوي على عدد غير محدود من Variants
- كل Variant: سعر التكلفة، سعر البيع، المصنعية الخاصة، الحالة

### 6 — دعم Variants للمنتجات ✅
- جدول `inventory_variants` مرتبط بكل فريم
- كل Variant: كود مستقل، كمية مستقلة، سعر مستقل
- واجهة accordion للعرض والإدارة

### 7 — صلاحيات مرنة للموظفين ✅
- 13 صلاحية مستقلة قابلة للتفعيل/التعطيل لكل موظف
- المسمى الوظيفي يُحمّل الصلاحيات الافتراضية مع إمكانية التعديل
- عرض الصلاحيات المفعّلة لكل موظف في الجدول

### 8 — Workshop Labor Cost ✅
- إعدادات المصنعية لكل نوع إطار في صفحة الإعدادات
- تُحتسب تلقائياً ضمن تكلفة الطلب
- التغيير يؤثر على الطلبات الجديدة فقط

### 9 — تثبيت الأسعار على الطلبات القديمة ✅
- جميع التكاليف تُخزَّن وقت إنشاء الطلب
- لا إعادة احتساب عند تغيير الأسعار مستقبلاً

### 10 — الهوية البصرية: ae Optics CRM ✅
- تغيير الاسم في كل مكان
- شعار ae مُدمج في Sidebar ومتغير المتصفح
- Favicon مخصص
- إزالة أيقونة النظارة الافتراضية

## التشغيل

```bash
npm install
npm run dev
```

## متغيرات البيئة

```env
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_anon_key
```

## قاعدة البيانات

شغّل `supabase_schema_v4.sql` في Supabase SQL Editor (يشمل جميع الجداول من الصفر).


### ملف البيئة
- تأكد أن اسم الملف هو `.env` في جذر المشروع، وليس `.env.txt`.
- ضع داخله `VITE_SUPABASE_URL` و `VITE_SUPABASE_ANON_KEY`.

### GitHub Pages Router
- تمّت إضافة صفحة 404 مخصصة مع استعادة المسار تلقائيًا، لذلك فتح أي Route مباشر أو Refresh على مسار فرعي سيعمل بدون HashRouter.
